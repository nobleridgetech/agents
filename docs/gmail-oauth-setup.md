# Gmail API OAuth Setup for Iris

This walkthrough sets up read-only Gmail API access for Iris against `nobleridgetech@gmail.com`.

Iris V1 must not send email. Do not grant Gmail send scopes for the first integration.

## What You Will Create

- A Google Cloud project.
- Gmail API enabled for that project.
- An OAuth consent screen in Testing mode.
- A Desktop OAuth client JSON file.
- A local token generated by approving access as `nobleridgetech@gmail.com`.

## Required Account

Use/sign in as:

```text
nobleridgetech@gmail.com
```

## Step 1: Create or Select a Google Cloud Project

1. Open: https://console.cloud.google.com/projectselector2/home/dashboard
2. Sign in with `nobleridgetech@gmail.com`.
3. Create a new project or select an existing Noble Ridge project.

Suggested project name:

```text
Noble Ridge Iris Gmail
```

## Step 2: Enable the Gmail API

1. Open: https://console.cloud.google.com/apis/library
2. Search for `Gmail API`.
3. Open `Gmail API`.
4. Click `Enable`.

Only Gmail API is needed for Iris email read/search. Do not enable extra APIs unless a future feature needs them.

## Step 3: Configure the OAuth Consent Screen

1. Open: https://console.cloud.google.com/auth/overview
2. Start configuring the OAuth consent screen.
3. User type:
   - If asked, choose `External` for a free Gmail account.
4. App name:

```text
Noble Ridge Iris
```

5. User support email:

```text
nobleridgetech@gmail.com
```

6. Developer contact email:

```text
nobleridgetech@gmail.com
```

7. Keep the app in `Testing` mode.
8. Add test user:

```text
nobleridgetech@gmail.com
```

If you see a page called `Audience`, add the test user there:
https://console.cloud.google.com/auth/audience

## Step 4: Choose Read-Only Scopes

When adding scopes, choose read-only Gmail scopes only.

Recommended for Iris V1:

```text
https://www.googleapis.com/auth/gmail.readonly
```

Do not add this scope for V1:

```text
https://www.googleapis.com/auth/gmail.send
```

Do not add broader Gmail modify/compose scopes unless the implementation is explicitly changed and approval gates are updated.

## Step 5: Create OAuth Client Credentials

1. Open: https://console.cloud.google.com/apis/credentials
2. Click `Create Credentials`.
3. Choose `OAuth client ID`.
4. Application type:

```text
Desktop app
```

5. Name:

```text
Iris Local OAuth Client
```

6. Click `Create`.
7. Download the JSON file.

Store it outside the repo. Suggested local path on Ubuntu 3:

```text
~/.config/noble-ridge-agents/google_client_secret.json
```

Never commit this file. Never paste the client secret into Discord, Obsidian, README, logs, or chat.

## Step 6: Give the JSON File Path to the Agent

Tell the agent the local file path in a sentence, for example:

```text
The Gmail OAuth client JSON is at /home/mark/.config/noble-ridge-agents/google_client_secret.json
```

Do not send the raw JSON contents unless explicitly needed.

## Step 7: Agent Runs Local OAuth Setup

The agent will then:

1. Validate that the JSON file exists.
2. Start an OAuth authorization flow.
3. Give you a Google authorization URL.
4. Ask you to approve access as `nobleridgetech@gmail.com`.
5. Ask you to paste back the redirected URL or authorization code.
6. Exchange the code for a local token.
7. Store the token outside the repo.
8. Verify Gmail read/search works.

Expected browser behavior:

- After approval, the browser may fail to load a localhost redirect URL.
- That is okay.
- Copy the full URL from the browser address bar and give it back to the agent.

## Step 8: Verification Target

The first real Gmail verification should only read/search messages, such as:

```text
Search for recent inbox messages newer_than:7d
Read one selected thread
Summarize it through Iris
Record the job in SQLite
Confirm no send scope or send call is used
```

## Troubleshooting

### Error 403: access_denied

Most likely cause: `nobleridgetech@gmail.com` was not added as a test user.

Fix:

1. Open: https://console.cloud.google.com/auth/audience
2. Add `nobleridgetech@gmail.com` as a test user.
3. Retry the OAuth flow.

### Gmail API Not Enabled

If the verification says the Gmail API is not enabled:

1. Open: https://console.cloud.google.com/apis/library/gmail.googleapis.com
2. Click `Enable`.
3. Retry.

### Insufficient Permission

The token was created without the required Gmail read-only scope.

Fix:

1. Revoke/delete the local token.
2. Re-run OAuth with `gmail.readonly`.

### App Not Verified Warning

Because this is an internal/local Testing-mode OAuth app, Google may show an unverified-app warning.

For this private local tool, continue only if:

- You are signed in as `nobleridgetech@gmail.com`.
- The app/project name is the Noble Ridge project you created.
- The scope is Gmail read-only.

## Security Rules

- Do not grant Gmail send scope for Iris V1.
- Do not put OAuth client JSON or token files in the repo.
- Do not paste secrets into Discord, Obsidian, README, or chat.
- Do not log token values.
- Validate credentials by presence/shape only.
- Keep the fake/local adapter tests passing before using the real Gmail adapter.
